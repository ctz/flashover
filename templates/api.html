Flashover API

Overview

A typical API use looks like:

  1. Submit SWF →
  2. Receive UUID →
  3. Poll for completion →
  4. Retrieve data

All requests are made over HTTP. Responses are JSON for metadata, or binary for resources. A UUID here identifies the result of decompiling your submitted SWF. They are allocated by flashover, and become invalid after two hours.
